k8s dashboard超详细部署教程

发表于 2021-06-18 | 分类于 运维
字数统计: 837 | 阅读时长 ≈ 3

1. 简介

在 Kubernetes 社区中，有一个很受欢迎的 Dashboard 项目，它可以给用户提供一个可视化的 Web 界面来查看当前集群的各种信息。用户可以用 Kubernetes Dashboard 部署容器化的应用、监控应用的状态、执行故障排查任务以及管理 Kubernetes 各种资源。

官方参考文档：
https://kubernetes.io/docs/tasks/access-application-cluster/web-ui-dashboard/

github项目地址：
https://github.com/kubernetes/dashboard

当前部署dashboard版本：v2.0.1，注意检查dashboard版本与kubernetes版本兼容性：
https://github.com/kubernetes/dashboard/releases

2. 部署

我们先把yaml文件下载下来

修改配置，可以在集群外部访问，找到kubernetes-dashboard的service，添加type: NodePort ，这里我指定的nodePort是30001

kubectl apply -f dashboard.yaml创建这个资源

查看一下服务是否启动

kubectl get pod -A -o wide

kubectl get svc -A

可以看到服务已经是正常启动了

3.访问认证

Dashboard 支持 Kubeconfig 和 Token 两种认证方式，我们这里选择Token认证方式登录

新建文件dashboard-adminuser.yaml

创建这个资源kubectl apply -f dashboard-adminuser.yaml

说明：上面创建了一个叫admin-user的服务账号，并放在kubernetes-dashboard 命名空间下，并将cluster-admin角色绑定到admin-user账户，这样admin-user账户就有了管理员的权限。默认情况下，kubeadm创建集群时已经创建了cluster-admin角色，我们直接绑定即可。

查看admin-user账户的token

可以看到有一个token，用于dashboard登录，刚才我们修改了文件，指定了30001端口，现在我们来访问一下，注意这里要用https://ip:nodePort,而且要用Firefox浏览器，Chrome 58以上就不行，如果一定要用Chrome，网上有相关文章

访问是这个样子，点击高级-接受风险并继续，输入上面的token登录。进去后可以看见k8s的控制台了
